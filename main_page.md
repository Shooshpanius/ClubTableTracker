Что изменить:
Секцию {clubs.map(club => { ... })} (строки ~776–825) — убрать аккордеон, заменить прямоугольными карточками.

Требования к карточке:

Карточка — прямоугольный блок. На десктопе (> 768px) — 2 карточки в ряд (CSS grid/flex с flex-wrap или grid-template-columns: 1fr 1fr). На мобильном — 1 в ряд.
Левая часть карточки (~1/3 ширины): логотип клуба (club.logoUrl). Изображение по центру, object-fit: contain. Если логотипа нет — заглушка (иконка/цвет). Клик по карточке — переход на страницу клуба (navigate('/club/' + club.id)), а не разворачивание аккордеона.
Правая часть карточки (~2/3 ширины): разделена на две колонки:
Средняя треть (левая часть правой зоны): текущие события клуба — это события (ClubEventItem), у которых startTime ≤ сейчас ≤ endTime. Показывать название и тип события. Если нет — «Нет активных событий».
Правая треть: предстоящие события — события, у которых startTime > сейчас, отсортированные по дате. Для каждого: дата + название. Показывать не более 3–5 ближайших.
Статус членства (✅/⏳/❌/🚫), кнопка «Подать заявку» — сохранить, разместить в правом нижнем углу карточки или в нижней части правой зоны.
Загрузка событий для карточек:
На главной нужно загрузить события для всех клубов при монтировании (или по мере рендера карточек). Использовать уже существующий endpoint /api/event/club/{clubId}. Хранить в state типа Record<number, ClubEventItem[]> (ключ — club.id).
Важно: это публичные данные для превью — загружать без авторизации (или с токеном если он есть, но не блокировать загрузку при его отсутствии).
