# Карта кампании — техническое задание

## Обзор

Для событий (`ClubEvent`) с типом `"Campaign"` реализовать интерактивную карту кампании.  
Карта состоит из блоков (территорий) и направленных связей между ними.  
Администратор клуба создаёт и редактирует карту; участники видят её только для чтения.

## Термины

| Термин | Значение |
|---|---|
| **N** («Максимальное влияние») | Количество столбцов внутри блока — максимально возможное влияние одной фракции |
| **M** («Фракции») | Список строк-фракций; количество записей = количество строк внутри блока |
| **Блок** | Прямоугольник N×M сегментов, представляющий территорию на карте |
| **Сегмент** | Одна ячейка (квадрат) внутри блока: (фракция i, уровень влияния j) |
| **Связь** | Направленная стрелка от одного блока к другому |

---

## Ограничения

- `N` и `M` (список фракций) задаются при создании/редактировании настроек карты.
- Изменять `N` и `M` можно **только если на карте ещё нет ни одного блока**.
- Список фракций хранится как упорядоченный массив строк; порядок важен (каждая строка = строка блока).
- Связи направленные (от-блока → к-блоку), допускают несколько связей между разными парами блоков.

## Модели данных (серверная сторона, C#)

### `CampaignMap`
```
Id            int     PK
EventId       int     FK → ClubEvent.Id, UNIQUE (1:1)
MaxInfluence  int     N — максимальное влияние (ширина блока)
Factions      string  JSON-массив строк, список фракций (M строк)
```

### `CampaignMapBlock`
```
Id      int     PK
MapId   int     FK → CampaignMap.Id
Title   string  Название территории (макс. 200 символов)
PosX    double  Координата X центра на канвасе
PosY    double  Координата Y центра на канвасе
```

### `CampaignMapBlockFaction`
```
Id             int  PK
BlockId        int  FK → CampaignMapBlock.Id
FactionIndex   int  Индекс фракции в массиве CampaignMap.Factions (0-based)
Influence      int  Текущее влияние (0 .. N)
```

### `CampaignMapLink`
```
Id           int  PK
MapId        int  FK → CampaignMap.Id
FromBlockId  int  FK → CampaignMapBlock.Id
ToBlockId    int  FK → CampaignMapBlock.Id
```

## API (серверная сторона)

Все эндпоинты требуют авторизации (`[Authorize]`).  
**AdminOrModerator** = пользователь является администратором или модератором клуба, которому принадлежит событие.

### `GET /api/campaign-map/{eventId}`
- Доступ: любой **член клуба**.
- Возвращает: `CampaignMap` с вложенными блоками (список `CampaignMapBlock` + `CampaignMapBlockFaction[]`) и связями `CampaignMapLink[]`.
- Если карта не создана → `404`.

### `POST /api/campaign-map/{eventId}`
- Доступ: **AdminOrModerator**.
- Тело: `{ maxInfluence, factions[] }`.
- Создаёт карту. Если карта уже существует → `409 Conflict`.

### `PUT /api/campaign-map/{eventId}/settings`
- Доступ: **AdminOrModerator**.
- Тело: `{ maxInfluence, factions[] }`.
- Обновляет параметры карты.
- Запрещено (`400 Bad Request`), если на карте уже есть хотя бы один блок.

### `POST /api/campaign-map/{eventId}/blocks`
- Доступ: **AdminOrModerator**.
- Тело: `{ title, posX, posY }`.
- Создаёт блок; возвращает созданный блок с `id`.

### `PUT /api/campaign-map/{eventId}/blocks/{blockId}`
- Доступ: **AdminOrModerator**.
- Тело: `{ title, posX, posY, factions[{ factionIndex, influence }] }`.
- Обновляет блок и значения влияния фракций.

### `DELETE /api/campaign-map/{eventId}/blocks/{blockId}`
- Доступ: **AdminOrModerator**.
- Удаляет блок и все его связи (`FromBlockId == blockId || ToBlockId == blockId`).

### `POST /api/campaign-map/{eventId}/links`
- Доступ: **AdminOrModerator**.
- Тело: `{ fromBlockId, toBlockId }`.
- Создаёт направленную связь.

### `DELETE /api/campaign-map/{eventId}/links/{linkId}`
- Доступ: **AdminOrModerator**.
- Удаляет связь.

## Клиентская часть (React/TypeScript)

Образец архитектуры — существующие компоненты `ClubMapEditor.tsx` и `ClubMap.tsx`.

### Новый компонент `CampaignMapEditor.tsx`
Полноэкранный редактор. Открывается из `ClubAdminPage.tsx` для события с типом `Campaign`.

**Функциональность:**
- Канвас с блоками и стрелками связей.
- **Блок** рисуется как сетка N×M ячеек с заголовком сверху.  
  Ячейка (строка = фракция i, столбец = уровень j) закрашена, если `influence ≥ j + 1`.
- Добавить блок: кнопка «Добавить блок» → блок появляется в центре канваса, перетаскивается мышью.
- Редактировать блок: клик по блоку → панель/диалог с полями: название, влияние каждой фракции (ползунок 0..N).
- Удалить блок: кнопка в панели редактирования блока.
- Добавить связь: режим «Соединить» → клик по блоку-источнику, затем по блоку-цели → рисует стрелку.
- Удалить связь: клик по стрелке → подтверждение удаления.
- Панель настроек карты: поля `maxInfluence` и редактируемый список `factions`.  
  Поля заблокированы (`disabled`), если на карте уже есть блоки.
- Изменения сохраняются через API при каждом действии (без черновика).

### Новый компонент `CampaignMapView.tsx`
Read-only версия карты. Открывается кнопкой «Карта кампании» в карточке события на `HomePage.tsx`.

**Функциональность:**
- Те же блоки и стрелки, без элементов редактирования.
- Блок при наведении (hover) показывает tooltip с названием и значениями влияния фракций.
- Поддерживает зум и скролл (адаптивность для мобильных экранов).
